乍一看以为是DFS，然后发现只要前面两个数确定了，其他都确定了。

当然你偏要用也是可以的，就是前面两层然后看是否可以一路走到底。

所以只用了两个循环来确定前两个数。

优化了一处就是第一个数只需查找到比num长度的一半或多1即可。
因为其他情况都不会满足至少3个数，且不能是以0开头的多位数。
这样可以省下部分时间。

还需要注意的是检验是否overflow。
此处用的是检验相加得到的数的str是否是num[start:]的开头。
即检验是否被截短了。
