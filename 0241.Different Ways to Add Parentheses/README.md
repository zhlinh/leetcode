第一版用的分治合并的思想。

自己都想到切分开最后再合并了，但就是没有想到用递归来解决。

把所遇到的符号的左右两边分别递归，返回两个数组，作为分治的结果。

然后用两个循环将该其进行合并。

分治会在最后输入只有一个数字的时候向上返回。
此处在一开始的时候判断，比较符合一般递归的思路。
也可在最后如果res为空(表示无运算符号)再返回该数字。

submit的结果为:
```
24 / 24 test cases passed.
Status: Accepted
Runtime: 56 ms
```
